# Spring Cashbook

## 프로젝트 소개

Spring Cashbook은 사용자가 일반 가계부처럼 거래를 입력하면, 시스템 내부에서 이를 복식부기 분개로 변환해 기록하는 미니 애플리케이션입니다.

사용자는 회계를 알 필요 없이 거래일, 거래 내용, 금액, 거래 유형, 결제 또는 입금 수단만 입력합니다. 

시스템은 이 입력을 바탕으로 현금, 카드미지급금, 미지급금의 변동을 계산하고 간단한 현금흐름과 재무상태 요약을 제공합니다.

이 프로젝트는 복잡한 회계 시스템이 아니라 개인 가계부 범위의 작은 애플리케이션입니다. 유형자산, 투자자산, 감가상각, 처분손익처럼 복잡한 회계 영역은 다루지 않고, 

개인의 수입과 소비, 카드 결제, 외상(빌림), 환불, 중고거래처럼 일상적인 거래를 회계적으로 일관되게 기록하는 데 집중합니다.

## 비즈니스 규칙

### 거래 입력

- 거래 유형은 수입, 지출, 중고거래, 환불로 구분합니다.
- 결제 또는 입금 수단은 현금, 카드, 외상/빌림으로 구분합니다.
- 계좌이체는 별도 수단으로 구분하지 않고 현금 거래로 취급합니다.
- 카드는 지출과 환불 거래에서만 사용할 수 있습니다.
- 외상/빌림은 지출 거래에서만 사용할 수 있습니다.

### 잔액

- 기초 잔액은 입력받지 않습니다.
- 현금과 부채 잔액은 앱에 기록된 거래 기준의 순변동으로 계산합니다.
- 현금 지출 시 앱 기록 기준 잔액이 부족하면 거래를 저장할 수 없습니다.
- 잔액이 부족한 경우 먼저 수입 또는 중고거래 입금을 기록하거나, 지출 수단을 외상/빌림으로 선택하도록 안내합니다.

### 수입과 지출

- 수입은 현금 잔액 증가로 반영합니다.
- 현금 결제와 계좌이체 지출은 거래일에 현금성 자산 감소로 반영합니다.
- 카드 결제 지출은 거래일에 현금 유출로 보지 않고 카드미지급금 증가로 반영합니다.
- 외상/빌림 지출은 거래일에 미지급금 증가로 반영합니다.

### 카드 대금과 외상/빌림 상환

- 카드 대금 납부 시 현금 잔액이 감소하고 카드미지급금이 감소합니다.
- 외상/빌림 상환 시 현금 잔액이 감소하고 미지급금이 감소합니다.

### 중고거래와 환불

- 중고거래는 일반 수입과 별도의 거래 유형으로 다룹니다.
- 이 프로젝트에서 소비 지출로 입력된 물건은 구매 시점에 비용 처리되며, 이후 중고거래 판매금은 개인 가계부 관점의 현금 유입으로 별도 기록합니다.
- 환불은 새로운 수입이 아니라 기존 지출의 취소로 처리합니다.

## 비즈니스 규칙 변경 메모 (2026.06.07 18:57)

### 환불과 취소

- 환불은 독립적인 거래 유형이 아니라 기존 거래 효과를 되돌리는 처리로 보는 것이 더 자연스럽습니다.
- 수입, 지출, 중고거래는 모두 취소될 수 있습니다.
- 취소 처리는 원 거래의 분개를 반대로 기록하여 기존 거래 효과를 되돌립니다.
- 따라서 환불 거래 유형은 제거하고, 기존 거래의 취소 처리로 기록합니다.
- 카드 대금 납부와 외상/빌림 상환은 거래 취소가 아니라 별도 상환 사건입니다.
- 카드 대금 납부와 외상/빌림 상환은 추후 별도 거래 유형 또는 별도 처리로 추가합니다.
